# Security Policy

## Supported Versions

| Version | Security support |
| --- | --- |
| `main` branch | Active development and fixes before the next release. |
| Latest PyPI release | Security fixes are prioritized. |
| Older releases | Best effort; upgrade to the latest release whenever possible. |

## Reporting a Vulnerability

Please do not open a public issue with exploit details.

Use the repository's private GitHub security advisory flow when it is available:

1. Open the repository on GitHub.
2. Go to **Security**.
3. Choose **Report a vulnerability**.

If private reporting is not enabled yet, open a minimal public issue asking for a private contact path, without sharing technical exploit details.

Helpful reports include:

- affected `quantum-circuit-drawer` version
- Python version and operating system
- installation command or extras used
- small reproduction steps
- expected impact and whether secrets, local files, or generated artifacts are involved

I aim to acknowledge valid reports within 7 days and publish a fix, mitigation, or status update as soon as practical for the severity.
